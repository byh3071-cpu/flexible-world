## 0. 한 줄 요약

> **"신은 시나리오를 쓰지 않았다. 당신은 이유 없이 이 세계에 던져졌을 뿐."**
> 정해진 운명·직업·튜토리얼이 없는 실존주의 샌드박스 커뮤니티 월드.

---

## 1. 비전 (Vision)

영감의 원천은 **서울관악지역자활센터 청년사업단 "플렉시블"** 의 실제 경험이다:
- 2024년 초창기 무법지대 → 규칙 건의·합의 → 갈등 → 시간이 지나 평화
- 이 과정을 **운영자 개입 없이** 자율적으로 일어나도록 디지털 공간에 옮기는 것이 목표
- 책임감 없는 책임감 / 자유의 고통 / 실존주의 / 자율성

장기 비전(현재 기술로는 어려움)은 **멀티 장르 통합 가상세계** — 광장에서 만나고, 문을 열면 2D 픽셀 마을, 다른 문은 FPS, 다른 문은 오픈월드. 현재 V18은 그 첫 단계인 **2D 픽셀 커뮤니티**.

## 2. 핵심 철학 (반드시 코드에 반영)

### 2-1. 개발자 = 이신론적 신
- 만든다: **물리 법칙·재료·도구**
- 만들지 않는다: **사회적 기능 (감옥·투표·추방·운영자 권한)**
- 예시: "감옥 시스템을 만들지 말고, 부술 수 없는 벽돌(Bedrock) + 자물쇠(Lock)만 던져줘라. 감옥을 짓든 집을 짓든 알아서 해라."

### 2-2. 절대 원칙: "나"를 증명하라
- 직업 선택창·성별 선택창 없음
- 캐릭터 상태는 행동으로만 결정 (`인벤토리 = 본질`)
- 예: 막대기를 줍는 순간 "생존자?", 누군가에게 말 거는 순간 "대화하는 자?", 공격 시 몸이 붉게, 도움 시 파랗게.

### 2-3. 강제 없는 책임
- 운영자가 제재하지 않음 → 유저 간 **평판 시스템**
- 평판 낮으면 특정 구역에서 자동 격리 (시스템이 조용히, 안내 없이)
- 자치구는 유저들이 직접 형성. A구역은 PK 가능, B구역은 대화만 가능, 모두 유저의 합의.

### 2-4. 피투성 (Thrownness)
- 시작 화면: 검은 화면 + 타자기 효과
  > "신은 시나리오를 쓰지 않았다.
  > 당신은 이유 없이 이 세계에 던져졌을 뿐.
  > 정해진 운명은 없다. 오직 당신의 선택만이 당신이 누구인지 증명할 것이다.
  > 자, 이제 이 고통스러운 자유 속에서 무엇이 될 텐가?"
- 즉시 랜덤 스폰, 튜토리얼 없음.

## 3. 3대 도구 (개발자가 제공해야 할 전부)

### 3-1. Matter — 무너뜨리고 쌓을 수 있는 권한
- 타일(Tile) 설치·파괴
- 감옥을 만들고 싶다면? 유저들이 벽돌을 둘러쌓아 물리적으로 가둔다.
- 갇힌 자는 곡괭이로 부수고 탈출.

### 3-2. Force — 물리적 힘과 체력
- 캐릭터 간 밀치기(Push)
- HP·공격
- 추방 버튼 없음. 다수가 절벽까지 밀어내야 함.

### 3-3. Communication — 기록과 전달
- 표지판·종이만 제공
- 투표 UI 없음
- 규칙을 만들고 싶다면? 표지판을 박고 직접 타이핑.
- 그 표지판을 다른 유저가 부수거나 내용을 바꿀 수도 있음.

## 4. 시뮬레이션 (예상 사회 진화 단계)

1. **초기 혼돈** — 만인에 대한 만인의 투쟁
2. **질서의 태동** — 힘센 유저들이 벽을 쌓고 안전지대 형성
3. **권력의 탄생** — 안전지대 입장료(조공) 요구
4. **혁명** — 착취당한 유저가 무기를 모아 벽을 무너뜨림

→ 운영자는 단 한 번도 개입하지 않음. 정치·경제·전쟁·철학이 자발적으로 발생.

## 6. 인프라 (Phase 0 — 무료 운영)

| 자원 | 도구 | 무료 한도 |
|---|---|---|
| 호스팅 | Vercel / GitHub Pages | 무제한 |
| 실시간 DB | Firebase Realtime DB (Spark) | 동시접속 ~100명, 1GB, 월 10GB 전송 |
| 이미지 | 정적 호스팅 분리 | — |
| 인증 | Firebase Anonymous Auth | 무제한 |

→ Firebase는 좌표·채팅 텍스트만, 이미지는 별도. 동시접속 500명 넘으면 월 5~10만원. 그 시점엔 BM이 이미 작동.
